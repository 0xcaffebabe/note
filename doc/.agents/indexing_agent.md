
背景：知识索引是本知识体系一种应用知识的视角，它可以站在某一角度（如面试八股文）来索引现有知识文档的知识。

用户输入：

```
帮我生成操作系统面试索引
```

## 步骤 1：读取相关内容

- [通过 SUMMARY.md 找到所有跟操作系统有关的文档]
- [读取所有操作系统相关的文档]


## 步骤2：系统输出

```
- 进程与线程的根本区别是什么？ -> [进程的本质](/操作系统/进程与线程.md)（看“为什么需要引入线程”部分）
- 虚拟内存解决了什么问题？ -> [虚拟内存的设计哲学](/操作系统/虚拟内存.md)
- 用户态切换到内核态有哪些方式？ -> [系统调用的代价](/操作系统/操作系统.md)
```

其中，每条索引应该有两部分组成：

1. 具体的问题（如：进程与线程的区别）
2. 解释该问题的根本原理（如：进程的本质，线程的本质），根本不应展开，而只是通过一个名称指向原理，通过绝对路径引用到具体的文档。

## 步骤3：写入生成的索引

生成的索引，应通过文件写入工具写入到 **知识索引** 目录下，如果要写入的文件不存在，应调用文件创建工具先创建待写入的文件。
