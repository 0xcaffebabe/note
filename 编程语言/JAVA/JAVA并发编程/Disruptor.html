Disruptor

дёҖгҖҒй—®йўҳиғҢжҷҜпјҡдёәд»Җд№ҲйңҖиҰҒ Disruptor иҝҷз§Қе№¶еҸ‘жЁЎеһӢ

еңЁй«ҳе№¶еҸ‘зі»з»ҹдёӯпјҢдј з»ҹзҡ„е№¶еҸ‘йҳҹеҲ—жЁЎеһӢпјҲеҰӮ BlockingQueueпјүжҷ®йҒҚйқўдёҙдёүдёӘж №жң¬жҖ§з“¶йўҲпјҡ

  1. еҶ…еӯҳеҲҶй…ҚдёҺ GC еҺӢеҠӣ

    • й“ҫиЎЁжҲ–иҠӮзӮ№еҜ№иұЎйў‘з№ҒеҲӣе»ә
    • еҜ№иұЎз”ҹе‘Ҫе‘ЁжңҹзҹӯпјҢи§ҰеҸ‘йў‘з№Ғ GC

  2. й”ҒдёҺдёҠдёӢж–ҮеҲҮжҚўжҲҗжң¬

    • й”Ғз«һдәүеҜјиҮҙзәҝзЁӢйҳ»еЎһ
    • дёҠдёӢж–ҮеҲҮжҚўз ҙеқҸ CPU Cache еұҖйғЁжҖ§

  3. CPU Cache жңӘиў«и§ҶдёәдёҖзӯүе…¬ж°‘

    • е№¶еҸ‘и®ҫи®ЎеҒңз•ҷеңЁвҖңзәҝзЁӢе®үе…ЁвҖқеұӮйқў
    • еҝҪз•Ҙ Cache LineгҖҒFalse SharingгҖҒжөҒж°ҙзәҝзӯүзЎ¬д»¶дәӢе®һ

Disruptor зҡ„жң¬иҙЁзӣ®ж Үпјҡ

еңЁдёҚеј•е…ҘеӨҚжқӮй”Ғз»“жһ„зҡ„еүҚжҸҗдёӢпјҢжһ„е»әдёҖз§Қйқўеҗ‘ CPU дёҺеҶ…еӯҳеұӮж¬Ўз»“жһ„зҡ„й«ҳеҗһеҗҗгҖҒдҪҺе»¶иҝҹе№¶еҸ‘ж•°жҚ®йҖҡйҒ“гҖӮ

дәҢгҖҒз¬¬дёҖжҖ§еҺҹзҗҶпјҡDisruptor зҡ„и®ҫи®Ўе“ІеӯҰ

Disruptor е№¶дёҚжҳҜвҖңжӣҙеҝ«зҡ„йҳҹеҲ—вҖқпјҢиҖҢжҳҜеҹәдәҺд»ҘдёӢз¬¬дёҖжҖ§еҺҹзҗҶжһ„е»әзҡ„е№¶еҸ‘дҪ“зі»гҖӮ

1. ж•°жҚ®иҝһз»ӯжҖ§дјҳе…ҲдәҺз»“жһ„зҒөжҙ»жҖ§

  • иҝһз»ӯеҶ…еӯҳпјҲж•°з»„пјүжҜ”зҰ»ж•ЈеҶ…еӯҳпјҲй“ҫиЎЁпјүжӣҙз¬ҰеҗҲ CPU и®ҝй—®жЁЎеһӢ
  • иҝһз»ӯи®ҝй—®еҸҜжңҖеӨ§еҢ– Cache Line е‘ҪдёӯзҺҮ

и®ҫи®Ўз»“и®әпјҡдҪҝз”Ёеӣәе®ҡй•ҝеәҰзҡ„зҺҜеҪўж•°з»„пјҲRingBufferпјүпјҢж”ҫејғеҠЁжҖҒз»“жһ„еёҰжқҘзҡ„зҒөжҙ»жҖ§пјҢжҚўеҸ–зЎ®е®ҡжҖ§зҡ„жҖ§иғҪгҖӮ

2. йЎәеәҸдёҖиҮҙжҖ§дјҳе…ҲдәҺдә’ж–ҘеҗҢжӯҘ

  • е№¶еҸ‘зҡ„жң¬иҙЁй—®йўҳдёҚжҳҜвҖңдә’ж–ҘвҖқпјҢиҖҢжҳҜвҖңйЎәеәҸвҖқ
  • еҸӘиҰҒз”ҹдә§дёҺж¶Ҳиҙ№еңЁ**еәҸеҸ·еӣ жһңе…ізі»**дёҠиҫҫжҲҗдёҖиҮҙпјҢе°ұдёҚйңҖиҰҒдј з»ҹж„Ҹд№үдёҠзҡ„й”Ғ

и®ҫи®Ўз»“и®әпјҡд»Ҙ SequenceпјҲйҖ’еўһеәҸеҸ·пјүдҪңдёәе№¶еҸ‘еҚҸи°ғзҡ„ж ёеҝғеҺҹиҜӯпјҢиҖҢдёҚжҳҜ LockгҖӮ

3. CPU Cache жҳҜе№¶еҸ‘зі»з»ҹзҡ„дёҖйғЁеҲҶ

  • Cache Line жҳҜ CPU еҸҜи§ҒжҖ§зҡ„жңҖе°ҸеҚ•дҪҚ
  • False Sharing дјҡеҜјиҮҙж— ж„Ҹд№үзҡ„ Cache жҠ–еҠЁ

и®ҫи®Ўз»“и®әпјҡйҖҡиҝҮ Cache Line PaddingпјҲеЎ«е……пјүжҳҫејҸйҒҝе…Қ False SharingпјҢдҪҝзәҝзЁӢй—ҙзҡ„зҠ¶жҖҒзңҹжӯЈзӢ¬з«ӢгҖӮ

дёүгҖҒж ёеҝғжһ¶жһ„жЁЎеһӢпјҡеҹәдәҺеәҸеҸ·зҡ„е№¶еҸ‘еҚҸдҪңдҪ“зі»

1. жһ¶жһ„жҠҪиұЎи§Ҷеӣҫ

Disruptor зҡ„ж ёеҝғжһ¶жһ„еҸҜд»ҘжҠҪиұЎдёәдёүзұ»и§’иүІпјҡ

  • **е…ұдә«ж•°жҚ®з»“жһ„**пјҡRingBuffer
  • **зҠ¶жҖҒжҺЁиҝӣеҷЁ**пјҡSequence
  • **еҚҸдҪңзӯ–з•Ҙ**пјҡWaitStrategy

е®ғд»¬е…ұеҗҢжһ„жҲҗдёҖдёӘеҹәдәҺеәҸеҸ·еӣ жһңе…ізі»зҡ„е№¶еҸ‘жү§иЎҢжЁЎеһӢгҖӮ

2. RingBufferпјҡе№¶еҸ‘зі»з»ҹзҡ„е…ұдә«еҶ…еӯҳе№ійқў

жң¬иҙЁе®ҡд№ү

RingBuffer жҳҜдёҖдёӘпјҡ

  • еӣәе®ҡеӨ§е°Ҹ
  • йў„еҲҶй…Қ
  • еҸҜйҮҚеӨҚдҪҝз”Ё

зҡ„еҫӘзҺҜж•°з»„гҖӮ

жһ¶жһ„д»·еҖј

  • ж¶ҲйҷӨиҝҗиЎҢжңҹеҶ…еӯҳеҲҶй…Қ вҶ’ йҷҚдҪҺ GC
  • еҲ©з”ЁеҶ…еӯҳеұҖйғЁжҖ§ вҶ’ жҸҗй«ҳ CPU Cache е‘Ҫдёӯ
  • йҖҡиҝҮеҸ–жЁЎдҪҚиҝҗз®—еҝ«йҖҹе®ҡдҪҚе…ғзҙ 

ж•°з»„й•ҝеәҰејәеҲ¶дёә 2вҒҝпјҢдҪҝдёӢж Үе®ҡдҪҚеҸҜйҖҡиҝҮдҪҚиҝҗз®—е®ҢжҲҗпјҢиҖҢйқһеҸ–жЁЎиҝҗз®—гҖӮ

3. Sequenceпјҡе№¶еҸ‘еӣ жһңе…ізі»зҡ„жҠҪиұЎ

Sequence зҡ„и§’иүІе®ҡдҪҚ

Sequence дёҚжҳҜз®ҖеҚ•зҡ„и®Ўж•°еҷЁпјҢиҖҢжҳҜпјҡ

жҸҸиҝ°вҖңдәӢд»¶еңЁе№¶еҸ‘зі»з»ҹдёӯжүҖеӨ„йҳ¶ж®өвҖқзҡ„зҠ¶жҖҒжҢҮй’ҲгҖӮ

еңЁзі»з»ҹдёӯиҮіе°‘еӯҳеңЁдёүзұ» Sequenceпјҡ

  • Cursorпјҡз”ҹдә§иҖ…е·ІеҸ‘еёғзҡ„жңҖеӨ§еәҸеҸ·
  • Producer Sequenceпјҡз”ҹдә§иҝӣеәҰ
  • Consumer Sequenceпјҡж¶Ҳиҙ№иҝӣеәҰ

ж ёеҝғжҖқжғі

  • жүҖжңүзәҝзЁӢеҸӘе…іеҝғвҖңжҲ‘иғҪеҗҰжҺЁиҝӣеҲ°жҹҗдёӘеәҸеҸ·вҖқ
  • е№¶еҸ‘еҶІзӘҒиў«иҪ¬еҢ–дёә**еәҸеҸ·еҸҜи§ҒжҖ§еҲӨж–ӯ**

4. ж— й”Ғзҡ„жң¬иҙЁпјҡCAS + жңүз•ҢжҺЁиҝӣ

Disruptor е№¶йқһе®Ңе…ЁвҖңж— еҗҢжӯҘвҖқпјҢиҖҢжҳҜпјҡ

  • дҪҝз”Ё CAS дҝқиҜҒеәҸеҸ·жҺЁиҝӣзҡ„еҺҹеӯҗжҖ§
  • дҪҝз”Ё RingBuffer зҡ„е®№йҮҸзәҰжқҹйҒҝе…Қж— йҷҗеҲ¶еҶҷе…Ҙ

жң¬иҙЁз»“и®әпјҡDisruptor е°Ҷе№¶еҸ‘жҺ§еҲ¶д»ҺвҖңж•°жҚ®дә’ж–ҘвҖқиҪ¬еҢ–дёәвҖңиҝӣеәҰеҚҸи°ғвҖқгҖӮ

еӣӣгҖҒCache Line дёҺ False Sharing жІ»зҗҶ

1. й—®йўҳжң¬иҙЁ

еҪ“еӨҡдёӘзәҝзЁӢйў‘з№Ғдҝ®ж”№дҪҚдәҺеҗҢдёҖ Cache Line зҡ„дёҚеҗҢеҸҳйҮҸж—¶пјҢдјҡдә§з”ҹ False SharingпјҢеҜјиҮҙжҖ§иғҪжҖҘеү§дёӢйҷҚгҖӮ

2. Disruptor зҡ„и§ЈеҶіж–№жЎҲ

  • дҪҝз”Ё Cache Line Padding
  • дҪҝз”Ё @Contended жҲ–жҳҫејҸ long еЎ«е……еӯ—ж®ө

йҖҡиҝҮдәәдёәжү©еӨ§еҸҳйҮҸй—ҙи·қпјҢдҪҝжҜҸдёӘзғӯзӮ№зҠ¶жҖҒзӢ¬еҚ дёҖдёӘ Cache LineгҖӮ

иҝҷдёҚжҳҜ Java жҠҖе·§пјҢиҖҢжҳҜе№¶еҸ‘зі»з»ҹеҝ…йЎ»йқўеҜ№зҡ„зЎ¬д»¶зҺ°е®һгҖӮ

дә”гҖҒз”ҹдә§иҖ…жЁЎеһӢпјҡе№¶еҸ‘еҶҷе…Ҙзҡ„жһ¶жһ„йҖүжӢ©

ProducerType зҡ„жһ¶жһ„еҗ«д№ү

жЁЎејҸе№¶еҸ‘еҒҮи®ҫжһ¶жһ„д»Јд»·йҖӮз”ЁеңәжҷҜ
SINGLEеҚ•зәҝзЁӢеҶҷе…ҘжңҖдҪҺжҳҺзЎ®еҚ•еҶҷеңәжҷҜ
MULTIеӨҡзәҝзЁӢеҶҷе…ҘCAS з«һдәүеӨҡз”ҹдә§иҖ…зі»з»ҹ

иҝҷжҳҜдёҖдёӘе…ёеһӢзҡ„вҖңз”ЁзәҰжқҹжҚўжҖ§иғҪвҖқзҡ„жһ¶жһ„еҶізӯ–зӮ№гҖӮ

е…ӯгҖҒWaitStrategyпјҡCPU дёҺе»¶иҝҹзҡ„еҚҡејҲзӯ–з•Ҙ

WaitStrategy е№¶йқһе®һзҺ°з»ҶиҠӮпјҢиҖҢжҳҜзі»з»ҹи°ғеәҰе“ІеӯҰгҖӮ

жҠҪиұЎз»ҙеәҰ

WaitStrategy еңЁд»ҘдёӢз»ҙеәҰдёҠеҒҡжқғиЎЎпјҡ

  • CPU еҚ з”ЁзҺҮ
  • е“Қеә”е»¶иҝҹ
  • зәҝзЁӢеҲҮжҚўжҲҗжң¬

еёёи§Ғзӯ–з•ҘеҲҶзұ»

  • **иҮӘж—ӢеһӢ**пјҡжһҒдҪҺе»¶иҝҹпјҢй«ҳ CPU еҚ з”Ё
  • **и®©жӯҘеһӢ**пјҡжҠҳдёӯж–№жЎҲ
  • **йҳ»еЎһеһӢ**пјҡдҪҺ CPUпјҢеҚ з”Ёй«ҳе»¶иҝҹ

йҖүжӢ© WaitStrategyпјҢжң¬иҙЁдёҠжҳҜеңЁйҖүжӢ©зі»з»ҹзҡ„жҖ§иғҪжҖ§ж јгҖӮ

дёғгҖҒејӮеёёеӨ„зҗҶпјҡе№¶еҸ‘зі»з»ҹзҡ„зЁіе®ҡжҖ§иҫ№з•Ң

ејӮеёёзҡ„жһ¶жһ„еҪұе“Қ

  • ејӮеёёе№¶дёҚеҸӘжҳҜдёҡеҠЎй—®йўҳ
  • еҸҜиғҪеҜјиҮҙ Sequence еҒңж»һпјҢиҝӣиҖҢеј•еҸ‘зі»з»ҹзә§йҳ»еЎһ

Disruptor зҡ„еӨ„зҗҶжЁЎеһӢ

  • е…ЁеұҖй»ҳи®ӨејӮеёёеӨ„зҗҶеҷЁ
  • й’ҲеҜ№зү№е®ҡж¶Ҳиҙ№иҖ…зҡ„е®ҡеҲ¶ејӮеёёзӯ–з•Ҙ

е№¶еҸ‘зі»з»ҹеҝ…йЎ»жҳҫејҸе®ҡд№үвҖңејӮеёёжҳҜеҗҰдёӯж–ӯж•°жҚ®жөҒвҖқгҖӮ

е…«гҖҒйҖӮз”ЁжҖ§дёҺйҖүеһӢиҫ№з•Ң

йҖӮеҗҲеңәжҷҜ

  • жһҒдҪҺе»¶иҝҹиҰҒжұӮ
  • й«ҳеҗһеҗҗгҖҒеҶ…еӯҳеҸҜжҺ§
  • дәӢд»¶еӨ„зҗҶйҖ»иҫ‘зӣёеҜ№з®ҖеҚ•

  • еӨҚжқӮйҳ»еЎһйҖ»иҫ‘
  • еҠЁжҖҒж¶Ҳиҙ№иҖ…жӢ“жү‘
